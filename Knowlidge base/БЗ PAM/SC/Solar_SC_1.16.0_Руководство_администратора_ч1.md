## 1. Общие сведения
Программный комплекс SafeConnect (далее — SafeConnect, Портал) предоставляет пользователю простой и безопасный способ доступа к удаленному ресурсу, если в этом возникает необходимость. Этот способ заключается в использовании веб-браузера с выходом на Портал, далее появляется возможность подключения к различным приложениям прямо из окна браузера, без использования дополнительного программного обеспечения.

SafeConnect позволяет исключить необходимость прямой публикации в сети интернет таких сервисов удаленного администрирования, как RDP, SSH, Telnet и др. для обеспечения доступа прямо к ресурсам внутри сети компании. При использовании ПК SafeConnect появляется возможность обеспечения централизованного, безопасного доступа к ресурсам компании, с возможностью применения функций SSO к разным ресурсам. При этом пользователь может подключаться к ресурсам с использованием аутентификационных данных, отличных от тех, которые он вводил на портале для авторизации, и при этом они ему не доступны. Доступ для пользователя обеспечивается только к тем ресурсам, к которым ему явным образом был разрешен доступ, другие ресурсы компании при этом недоступны. Они просто не публикуются у него на странице.

Портал может использоваться самостоятельно, а также может быть использован совместно с системой контроля привилегированных пользователей SafeInspect для обеспечения возможности записи действий удаленных пользователей и других функций по управлению доступом к критичным ресурсам.

### 1.1. Возможности SafeConnect
SafeConnect выполняет следующие задачи	:
- Создание Портала доступа и публикации на нем ресурсов компании для удаленных пользователей. Каждому пользователю будут доступны только те ресурсы, которые ему явно были назначены;
- Обеспечение строгой аутентификации пользователей, а также разграничение доступа к публикуемым ресурсам;
- Обеспечение возможности категорирования ресурсов и пользователей по разным уровням доступа (критические ресурсы, ресурсы с персональными данными (Федеральный закон от 27.07.2006 г. № 152-ФЗ "О персональных данных" (далее — ФЗ–152)), General Data Protection Regulation (далее — GDPR)). А также обеспечение строгого мандатного доступа к разным категориям ресурсов в соответствии с допуском пользователей к таким категориям ресурсов;
- Обеспечение функций SSO (пользователь однократно авторизуется на Портале, затем подключается к ресурсам без введения пароля);
- Обеспечение подмены пользователей при доступе к ресурсам (при подключении через Портал, доступ к реальным ресурсам может осуществляться под другими учетными данными с автоматической их подменой);
- Обеспечение одноразового доступа к ресурсам, доступа по расписанию, с возможностью установки количества доступов и других параметров соединений;
- Обеспечение возможности постановки задач пользователям, что конкретно в каждом из их доступов они должны сделать (внутренняя тикетная система);
- Возможность создания Портала самообслуживания. Возможность создания групп пользователей с разными правами, где отдельные пользователи могут иметь права на создание тикетов и доступов к ресурсам, управляя, таким образом, доступом для своей группы пользователей и ресурсов;
- Интеграция с ПК SafeInspect для обеспечения эффективного контроля действий пользователей.

### 1.2. Логика работы SafeConnect
SafeConnect выполнен в виде шлюза, где обеспечивается возможность публикации приложений, обеспечивается строгая аутентификация и авторизация пользователей, с возможностью контроля подключений пользователей к ресурсам.

Для эффективного управления доступами и изменениями в SafeConnect существует специализированная тикетная система. Таким образом, каждый доступ обязательно получает свой тикет, и появляется возможность обеспечения диалога с администраторами ресурсов и получения необходимых заданий для пользователей с тем, чтобы они могли уточнить, что именно они могут делать в рамках предоставляемого доступа или какое задание необходимо выполнить в рамках этого доступа.

**Рис. 1.2.** Описание правил доступа

Все общение между администраторами Портала и пользователями может осуществляться через тикетную систему SafeConnect, исключая необходимость звонить администраторам и просить сделать какой-либо доступ и другие действия. Что сильно облегчает работу компании с удаленными пользователями в целом.

Описание правил доступа состоит из нескольких компонентов: собственно ресурсы и аккаунты для доступа к этим ресурсам, а также задания и политика доступа. Схематично это отражено на рисунке (Рис. 1.2).

Каждый пользователь может создать запрос на доступ к ресурсу как для себя, так и для своей группы пользователей, или для другого пользователя. Администраторы, предоставляя доступ к ресурсу, также создают задание (тикет) для пользователя (группы пользователей), в котором кратко описывают для чего этот доступ. Далее к этому заданию добавляются необходимые ресурсы, аккаунты, с помощью которых пользователи будут осуществлять доступ, политики, которые регламентируют когда и каким образом этот доступ должен осуществляться.

Далее при необходимости задание утверждается (согласовывается) другими администраторами (пользователями с такими правами), доступ автоматически создается и появляется на странице Портала пользователя.

## 2. Условия выполнения
Программный комплекс SafeConnect может быть установлен как на аппаратное устройство (компьютер), так и на виртуальное устройство (виртуальную машину). В этом разделе приведены системные требования для установки на виртуальное устройство.

### 2.1. Системные требования к виртуальному устройству
Для установки SafeConnect в качестве виртуальной среды рекомендуется использовать:
- Proxmox Virtual Environment (версии 8.2.3, 8.2.7);
- VMware ESXi (версия 7.0);
- VMware Workstation (версия 17).

**Табл. 2.1.** Минимальные системные требования к виртуальной машине VMware
| Значение | Спецификация |
|---|---|
| 2 GB или более | Память |
| 20 GB или более | Размер диска |
| Тип: Linux | Тип гостевой операционной системы, установленной на виртуальную машину |
| Версия: Astra Linux Special Edition 1.7 (64-bit) с последними обновлениями безопасности | Операционная система |
| Минимально 1 сетевой адаптер | Сетевое оборудование |

## 3. Работа с программой

### 3.1. Запуск программного комплекса SafeConnect и первоначальная настройка
Для запуска программного комплекса SafeConnect в веб-браузере наберите IP-адрес, который был введен во время инсталляции. После этого появится диалоговое окно **Создание суперадминистратора** с приглашением ввести логин и пароль суперадминистратора.

**Рис. 3.1.** Создание суперадминистратора

Откроется главное окно программного комплекса SafeConnect. Для первоначального входа в систему нужно ввести:
- логин суперадминистратора,
- e-mail,
- пароль,
- повторить пароль.

Дополнительно есть возможность выбора темы (с помощью переключателя в верхнем меню).

**Рис. 3.2.** Выбор темы

Для ввода данных нажать кнопку **Сохранить**.

Также реализована возможность восстановления системы из резервной копии сразу после установки. Для этого:
- выбрать файл резервной копии по кнопке **Выберите файл**;
- ввести пароль, назначенный при создании резервной копии;
- при необходимости включить/установить флажок **Восстановить системную конфигурацию**;
> **Примечание:** Настройки сети не восстанавливаются, если резервная копия была сделана на другой машине.
- нажать кнопку **Восстановить из файла**.

После сохранения первоначальных настроек (после восстановления системы из резервной копии) откроется главное окно программного комплекса SafeConnect.

**Рис. 3.3.** Главное окно SafeConnect

### 3.3. Главное меню
Главное меню обеспечивает доступ ко всем разделам работы SafeConnect и позволяет производить все необходимые настройки.

Главное меню состоит из следующих разделов:
- **Главная** — отображает ресурсы, назначенные текущему пользователю;
- **Сессии** — отображает информацию о доступе пользователей к ресурсам;
- **Ресурсы** — раздел для настройки и управления доступными ресурсами;
- **Учетные записи** — раздел управления учетными записями пользователей;
- **Политики** — раздел управления политиками доступа;
- **Пользователи** — раздел управления списком пользователей Портала;
- **Группы** — раздел управления группами пользователей;
- **Роли** — раздел управления списком ролей пользователей;
- **Настройки** — раздел содержит страницы с настройками: Лицензия, Безопасность, Интеграция с внешними системами, Настройки SMTP, Пользовательское соглашение, Репликация, Серверы LDAP, Серверы RADIUS, Сетевые настройки, Брендирование, Резервное копирование и восстановление, Система;
- **Задания** — раздел позволяет создавать задания для пользователей Портала.

### 3.4. Раздел «Главная»
Вкладка **Главная** является домашней страницей SafeConnect и открывается после авторизации пользователя.

В рабочей области отображаются все доступные данному пользователю ресурсы и разрешенные доступы. Чтобы просмотреть информацию по конкретному ресурсу, следует кликнуть на него мышкой.

По каждому ресурсу отображается следующая информация:
- **Статус** — отображается статус ресурса (доступен для подключения или нет);
- **Тип авторизации** — способ авторизации пользователя;
- **Учетная запись** — учетная запись пользователя, кому дан доступ;
- **Подключения** — общее количество подключений (`∞` — без ограничений);
- **Длительность сессии** — максимальная длительность сессии (`∞` — нет ограничений по длительности);
- **Время доступа** — время, когда разрешен доступ пользователю к данному ресурсу (значок бесконечности `∞` — разрешен доступ без ограничений по времени);
- **Следующая сессия через** — промежуток времени до следующей сессии;
- **Подтверждение** — индикатор необходимости дополнительного подтверждения доступа;
- **Задание** — задание, на основании которого выдан доступ.

Для нового запроса доступа к данному ресурсу следует установить переключатель в начале строки в положение «активен» и нажать кнопку **Запросить доступ**.

Над списком ресурсов расположено поле для контекстного поиска нужного ресурса. Начать вводить в поле поиска название ресурса и в списке останутся только те ресурсы, в названии которых есть введенное значение.

### 3.5. Раздел «Сессии»
В рабочей области раздела **Сессии** расположена таблица, в которой отображается информация по доступу пользователей SafeConnect к ресурсам. По каждому доступу отображается следующая информация:
- **Пользователь** — имя пользователя, осуществившего доступ к ресурсу;
- **Ресурс** — название ресурса, на который был доступ;
- **Протокол** — протокол, по которому был назначен доступ;
- **Авторизация** — тип авторизации и учетная запись;
- **Задание** — задание, на основании которого был выполнен доступ;
- **Дата создания**;
- **Начало** — дата и время начала доступа к ресурсу;
- **Окончание** — дата и время окончания пользовательской сессии;
- **Соединение** — статус подключения и длительность сессии.

**Рис. 3.13.** Раздел «Сессии»

#### 3.5.1. Статистика и поиск
Для просмотра статистики необходимо перейти в раздел **Сессии** и нажать кнопку **Статистика**.

**Рис. 3.17.** Расположение кнопки «Статистика»

Далее для просмотра станут доступны статистика и информация по подключениям пользователя.

Настроить параметры отображаемой статистики можно следующим образом:
- **Выбор отображаемых сессий:** Все сессии; Установленные соединения; Сейчас онлайн.
- **Установка временного периода:** Выбор периода из списка (Сегодня, Вчера, На этой неделе, В этом месяце, В этом году) или выбор периода вручную.
- **Установка времени:** Чтобы добавить временной диапазон, необходимо воспользоваться временными шкалами и установить время начала диапазона и время окончания диапазона.
- **Группировка мультисессий:** Для группировки сессий необходимо установить флажок **Группировать мультисессии**. Для разгруппировки необходимо снять данный флажок.
- **Выбор отображаемой информации:** Количество сессий / Время онлайн. По пользователям / По ресурсам / По учётным записям.

В разделе **Статистика** доступна функция поиска. Для отображения искомой информации необходимо ввести значение в поле поиска. Также имеется возможность уточнить поиск:
- Искать в имени пользователя;
- Искать в названии ресурса;
- Искать в названии учетной записи.

Для сохранения графика сессий необходимо воспользоваться кнопкой загрузки изображения. В результате на локальное хранилище пользователя будет скачан файл-изображение графика в формате `*.PNG`.

Рядом с графиком расположены области с информацией об учетных записях и пользователях. В данной области есть возможность сортировки по столбцам. Для включения сортировки нажать на название столбца.

Также реализованы следующие возможности:
- при помощи кнопок скрыть или отобразить строчки с общим количеством записей (пользователей, ресурсов или учетных записей). Это позволяет отобразить или скрыть общее число суммы всех записей.
> **Примечание:** У отдельных пользователей, ресурсов и учетных записей отсутствует возможность включения и выключения отображения.
- окрасить записи с текущей страницы.

Для выхода из просмотра статистики нажать **Скрыть статистику**.

##### 3.6.1.1. Добавление ресурсов из файла
В SafeConnect имеется возможность загрузить ресурсы на портал из файла в формате `*.csv`. Для этого следует нажать кнопку **Добавить из файла**, выбрать нужный файл и нажать кнопку **Открыть**. Ресурсы из файла отобразятся в списке ресурсов.

**Табл. 3.1.** Пример написания файла «Ресурсы» для загрузки
| options | critical | fz152 | gdpr | description | port | protocol | address | name |
|---|---|---|---|---|---|---|---|---|
| false | false | false | false | rdp res desc | 3389 | rdp | 192.168.56.101 | importedres #1 RDP |

Обозначения полей для загрузки:
- `Name` — название ресурса;
- `Address` — IP–адрес ресурса;
- `Protocol` — протокол (RDP/RDP App/RDS/SSH/HTTP/HTTRS/Telnet);
- `Port` — порт;
- `Description` — описание;
- `Gdpr` — ресурс с данными, подпадающими под GDPR (`true` — установить флажок, `false` — снять флажок);
- `fz152` — ресурс с данными, подпадающими под ФЗ–152 (`true` — установить флажок, `false` — снять флажок);
- `critical` — критический ресурс инфраструктуры (`true` — установить флажок, `false` — снять флажок);
- `options` — опциональное поле, может содержать дополнительные параметры (например, черный список, поля логин и пароль и т.д.).

##### 3.6.1.2. Возможность работы с несколькими вкладками в HTTP/HTTPs-сессии
В SafeConnect реализована поддержка работы с несколькими вкладками в рамках одного защищенного подключения к веб-ресурсам. Функционал позволяет пользователю одновременно работать в нескольких вкладках без необходимости инициировать новые подключения к ресурсу.

**Механизм работы:**
- При запуске подключения к ресурсу система автоматически открывает интерфейс с поддержкой вкладок.
- Допускается одновременное открытие и существование двух вкладок в рамках одной пользовательской сессии.
- Переключение между вкладками осуществляется только с помощью стандартных сочетаний клавиш:

**Табл. 3.2.** Комбинации клавиш для переключения вкладок
| Действие | Комбинация клавиш | ОС |
|---|---|---|
| Переход к следующей вкладке | `Ctrl + Tab` / `Ctrl + PgDn` | Windows/ Linux |
| Переход к предыдущей вкладке | `Ctrl + Shift + Tab` / `Ctrl + PgUp` | Windows/ Linux |
| Переход к вкладке по номеру | `Ctrl + 1-2` | Windows/ Linux |
| Переключение между вкладками | `Cmd+ Option+ Стрелка вправо/влево` | macOS |
| Переход к вкладке по номеру | `Cmd+ 1-2` | macOS |

**Настройка доступа:**
- Возможность использования нескольких вкладок настраивается администратором в параметрах конкретного HTTP/HTTPS ресурса. По умолчанию поведение определяется политикой безопасности ресурса.
- Конфигурация выполняется в карточке ресурса SafeConnect: параметры ресурса HTTP/HTTPS, установить флажок **Двойное открытие вкладок при подключении**. По умолчанию данный флажок снят.
- При включении опции система устанавливает жесткое ограничение — максимально возможное количество вкладок в одной сессии равно двум.

**Интеграция с безопасностью и аудитом:**
- Интеграция SafeInspect и SafeConnect: все действия во всех вкладках фиксируются в системе SafeInspect. Сессии отображаются в отчетах, статистике и журналах аудита наравне с обычными подключениями.
- Передача данных: полноценная поддержка работы с буфером обмена, а также загрузка и скачивание файлов доступны во всех открытых вкладках сессии.
- Подтверждение доступа: если подключение требует подтверждения (например, по заданию на доступ), оно запрашивается только один раз при инициализации сессии. Открытие двух вкладок в рамках этого соединения повторного подтверждения не требует.

##### 3.6.1.3. Загрузка и скачивание файлов на HTTPS-ресурсе
В SafeConnect реализована возможность прямого обмена данными между хостовой машиной пользователя и целевым веб-ресурсом при подключении через SafeConnect. Ранее работа с ресурсом была ограничена только просмотром репозитория, теперь поддерживаются операции с файлами.

> **Внимание!** Все файлы, загруженные на ресурс или скачанные с него через встроенный файловый менеджер, являются временными. Они автоматически удаляются после завершения работы и отключения пользователя. При следующем подключении к ресурсу файловый менеджер будет пуст (история файлов не сохраняется).

Пользователи, подключающиеся к защищенным HTTPS–ресурсам, теперь могут выполнять стандартные действия:
- Загрузка файлов с локального компьютера на удаленный веб-ресурс.
- Скачивание файлов с ресурса на хостовую машину пользователя.

Для использования данного функционала, он должен быть явно разрешен администратором в настройках ролей SafeConnect (раздел **Роли**). Функционал обеспечен на уровне Xrdp-обработчика, обеспечивая прозрачную передачу данных внутри сессии.

#### 3.6.2. Редактирование ресурса
Чтобы внести изменения в настройки ресурса, следует перейти на вкладку **Ресурсы**, нажать на значок редактирования в строке редактируемого ресурса и в открывшемся окне с данными ресурса внести необходимые изменения (описание полей см. в разделе 3.6.1), проверить доступность ресурса, и нажать кнопку **Сохранить**.

#### 3.6.3. Удаление ресурса
Для удаления ресурса следует перейти на вкладку **Ресурсы**, нажать на значок «крестик» в строке удаляемого ресурса, подтвердить удаление, нажав кнопку **ОК** в предупреждающем сообщении.
