## 3. Работа с программой (продолжение)

### 3.7. Раздел «Учетные записи»
Раздел **Учетные записи** предназначен для управления учетными данными, используемыми для подключения к публикуемым ресурсам. В отличие от учетных записей пользователей Портала, учетные записи в этом разделе хранят данные для авторизации на целевых системах (Windows, Linux, веб-сервисы и др.).

**Рис. 3.17.** Раздел «Учетные записи»

По каждой учетной записи отображается:
- **Название** — идентификатор учетной записи в системе;
- **Тип** — Windows, Linux, LDAP, Generic, HTTP Basic;
- **Статус** — активна/неактивна;
- **Последнее использование** — дата и время последнего подключения;
- **Дата создания**.

Для добавления новой учетной записи нажмите кнопку **Добавить**, выберите тип и заполните форму:
- **Логин** — имя пользователя для целевой системы;
- **Пароль** — вводится в зашифрованном виде;
- **Домен** (для Windows/LDAP) — имя домена или контроллера;
- **Описание** — необязательное поле для внутреннего учета.

> **Примечание:** Пароли хранятся в зашифрованном виде с использованием алгоритма AES-256. Прямой доступ к исходным значениям невозможен даже для суперадминистратора.

Для редактирования или удаления учетной записи используйте соответствующие значки в строке записи. При удалении учетная запись автоматически исключается из всех активных заданий и политик.

### 3.8. Раздел «Политики»
Раздел **Политики** позволяет настраивать правила доступа, ограничения сессий и параметры безопасности для ресурсов и пользователей.

**Рис. 3.18.** Раздел «Политики»

Каждая политика содержит:
- **Название и описание**;
- **Время доступа** — расписание (ежедневно, по будням, кастомный график);
- **Ограничение по количеству подключений** — одноразовый, лимитированный или без ограничений (`∞`);
- **Максимальная длительность сессии** — от 15 минут до 24 часов;
- **Требование подтверждения** — необходимость одобрения администратором перед подключением;
- **Протоколы шифрования** — принудительное использование TLS 1.2/1.3 для веб-ресурсов;
- **Ограничение буфера обмена и передачи файлов** — включено/выключено.

Для создания политики нажмите **Добавить**, настройте параметры и привяжите её к ресурсам или группам пользователей. Политики применяются каскадно: если к пользователю привязано несколько политик, применяются наиболее строгие ограничения.

### 3.9. Раздел «Пользователи»
Раздел **Пользователи** содержит реестр всех учетных записей, имеющих доступ к Порталу SafeConnect.

**Рис. 3.19.** Раздел «Пользователи»

Информация по пользователю:
- **ФИО / Логин**;
- **E-mail**;
- **Статус** — активен/заблокирован;
- **Группы** — привязанные группы пользователей;
- **Роли** — назначенные роли доступа;
- **Дата последнего входа**;
- **Количество активных сессий**.

Для добавления пользователя нажмите **Добавить**, укажите логин, e-mail, пароль (или включите сброс по ссылке), назначьте группу и роль. При блокировке пользователя все его активные сессии завершаются принудительно.

### 3.10. Раздел «Группы»
Раздел **Группы** позволяет объединять пользователей по функциональным признакам (отделы, проекты, уровни допуска) для упрощения управления доступами.

**Рис. 3.20.** Раздел «Группы»

Каждая группа содержит:
- **Название и описание**;
- **Список пользователей**;
- **Привязанные ресурсы и политики**;
- **Уровень допуска** — Общий, ФЗ-152, GDPR, Критический.

При изменении состава группы или привязанных политик изменения применяются автоматически ко всем членам группы. Группы поддерживают вложенность (группа может быть членом другой группы).

### 3.11. Раздел «Роли»
Раздел **Роли** определяет права пользователей внутри самого Портала SafeConnect (не путать с правами доступа к ресурсам).

**Рис. 3.21.** Раздел «Роли»

Стандартные роли:
- **Суперадминистратор** — полный доступ ко всем разделам и настройкам;
- **Администратор ресурсов** — управление ресурсами, учетными записями, политиками;
- **Администратор пользователей** — управление пользователями, группами, ролями;
- **Оператор тикетов** — создание, редактирование и завершение заданий;
- **Пользователь** — базовый доступ к Порталу и разрешенным ресурсам.

Для создания пользовательской роли нажмите **Добавить**, выберите доступные разделы и действия (просмотр, создание, редактирование, удаление). Роли применяются к пользователям или группам в разделе **Пользователи**.

### 3.12. Раздел «Настройки»
Раздел **Настройки** содержит глобальные параметры системы. Доступен только пользователям с ролью **Суперадминистратор**.

**Рис. 3.22.** Раздел «Настройки»

Основные подразделы:
- **Лицензия** — активация, проверка статуса, отображение ограничений;
- **Безопасность** — политика паролей, блокировка после N неудачных попыток, принудительная смена пароля, настройка 2FA;
- **Интеграция с внешними системами** — LDAP, RADIUS, SAML/OIDC, SafeInspect;
- **Настройки SMTP** — параметры почтового сервера для уведомлений и сброса паролей;
- **Пользовательское соглашение** — текст, отображаемый при первом входе;
- **Репликация** — настройка мастер-слейв архитектуры для отказоустойчивости;
- **Серверы LDAP / RADIUS** — параметры подключения и тестирование связи;
- **Сетевые настройки** — IP-адреса, порты, прокси, DNS;
- **Брендирование** — логотип, цвета, название портала, favicon;
- **Резервное копирование и восстановление** — расписание, хранилище, ручное создание копий;
- **Система** — версия, статус служб, перезапуск компонентов, очистка кэша.

### 3.13. Раздел «Задания»
Раздел **Задания** (тикетная система) используется для согласования, выдачи и контроля доступа к ресурсам.

**Рис. 3.23.** Раздел «Задания»

Каждое задание содержит:
- **Номер и статус** — Черновик, На согласовании, Активно, Завершено, Отклонено;
- **Исполнители** — пользователи или группы;
- **Ресурсы и учетные записи** — привязанные элементы доступа;
- **Политика** — правила подключения;
- **Описание задачи** — что должен сделать пользователь в рамках доступа;
- **Срок действия** — дата начала и окончания;
- **История действий** — аудит изменений и комментариев.

Для создания задания нажмите **Добавить**, заполните форму, выберите исполнителей и привяжите ресурсы. После отправки на согласование задание переходит в статус **На согласовании**. После утверждения доступ автоматически публикуется на странице пользователя.

### 3.14. Раздел «Аудит и журналы»
Раздел **Аудит и журналы** хранит детальные записи о действиях пользователей и администраторов.

**Рис. 3.24.** Раздел «Аудит и журналы»

Журналы включают:
- **Аутентификация** — успешные/неудачные входы в Портал;
- **Подключения к ресурсам** — начало/окончание сессий, используемые протоколы;
- **Изменения конфигурации** — создание/редактирование ресурсов, политик, пользователей;
- **Действия в тикетах** — создание, согласование, завершение заданий;
- **Системные события** — ошибки, перезапуски, резервное копирование.

Поддерживается фильтрация по дате, пользователю, типу события, ресурсу. Экспорт в формате `*.CSV` или `*.JSON` доступен по кнопке **Скачать**.

### 3.15. Раздел «Резервное копирование и восстановление»
Раздел позволяет создавать резервные копии конфигурации SafeConnect и восстанавливать систему при сбоях.

**Рис. 3.25.** Раздел «Резервное копирование»

Возможности:
- **Ручное создание копии** — кнопка **Создать резервную копию**;
- **Автоматическое расписание** — ежедневно, еженедельно, по кастомному графику;
- **Хранилище** — локальный диск или удаленный SFTP/S3-совместимый сервер;
- **Шифрование архива** — AES-256 с паролем;
- **Восстановление** — загрузка файла, ввод пароля, выбор компонентов для восстановления.

> **Внимание:** Резервная копия не включает данные сессий и временные файлы. Восстановление требует перезапуска служб SafeConnect.

### 3.16. Раздел «Обновление системы»
Раздел **Обновление системы** предназначен для установки патчей и новых версий SafeConnect.

**Рис. 3.26.** Раздел «Обновление системы»

Процесс обновления:
1. Проверка доступных обновлений кнопкой **Проверить обновления**;
2. Скачивание пакета и проверка цифровой подписи;
3. Создание автоматической резервной копии перед установкой;
4. Применение обновления с перезапуском служб;
5. Проверка целостности и статуса компонентов.

Обновления применяются без потери данных. При критических ошибках система автоматически откатывается к предыдущей версии.

## 4. Интеграция с внешними системами

### 4.1. Интеграция с LDAP/Active Directory
SafeConnect поддерживает аутентификацию пользователей через внешние каталоги. Настройка выполняется в разделе **Настройки → Серверы LDAP**.

Параметры подключения:
- **Адрес сервера** — IP или FQDN контроллера домена;
- **Порт** — 389 (LDAP) или 636 (LDAPS);
- **Базовая DN** — корневая область поиска пользователей;
- **Фильтр поиска** — `(objectClass=user)` или кастомный;
- **Учетная запись связи** — DN и пароль для привязки к каталогу;
- **Тест подключения** — проверка доступности и корректности настроек.

После настройки пользователи могут авторизовываться в Портале с учетными данными домена. Синхронизация групп выполняется по расписанию или вручную.

### 4.2. Интеграция с RADIUS
Для двухфакторной аутентификации или интеграции с корпоративными системами безопасности поддерживается протокол RADIUS.

Настройка в разделе **Настройки → Серверы RADIUS**:
- **Адрес сервера**, **Порт** (по умолчанию 1812);
- **Shared Secret** — общий ключ шифрования;
- **Тип аутентификации** — PAP, CHAP, MS-CHAPv2;
- **Тест подключения** — проверка обмена пакетами.

RADIUS используется как дополнительный этап проверки после ввода логина/пароля на Портале.

### 4.3. Интеграция с SafeInspect
SafeConnect интегрируется с ПК SafeInspect для записи действий пользователей, контроля привилегированных сессий и аудита.

**Рис. 4.1.** Схема интеграции SafeConnect + SafeInspect

Настройка выполняется в разделе **Настройки → Интеграция с внешними системами**:
- **Адрес сервера SafeInspect**;
- **API-ключ** — генерируется в консоли SafeInspect;
- **Режим записи** — полная сессия, только критические ресурсы, по расписанию;
- **Передача метаданных** — логин, IP, ресурс, время, задание.

Все сессии, инициированные через SafeConnect, автоматически передаются в SafeInspect. Записи доступны в виде видеопотока, текстового лога или скриншотов.

## 5. Безопасность и соответствие стандартам

### 5.1. Защита данных и шифрование
SafeConnect обеспечивает многоуровневую защиту:
- **Транспортное шифрование** — TLS 1.2/1.3 для всех веб-соединений;
- **Шифрование данных в покое** — AES-256 для паролей, конфигураций, резервных копий;
- **Изоляция сессий** — каждая сессия выполняется в отдельном контейнере без доступа к хостовой системе;
- **Защита от CSRF/XSS** — встроенные фильтры и валидация входных данных.

### 5.2. Соответствие ФЗ-152 и GDPR
Система поддерживает маркировку ресурсов, содержащих персональные данные:
- Флажки **ФЗ-152** и **GDPR** в карточке ресурса;
- Автоматическое применение усиленных политик (ограничение буфера обмена, запрет скачивания, обязательное подтверждение);
- Журналы аудита с фиксацией всех действий с помеченными ресурсами;
- Возможность экспорта отчетов для контрольных органов.

### 5.3. Мандатный контроль доступа
SafeConnect реализует мандатную модель безопасности:
- Пользователи получают **уровень допуска** (Общий, ФЗ-152, GDPR, Критический);
- Ресурсы маркируются соответствующим уровнем;
- Подключение возможно только при совпадении или превышении допуска пользователя;
- Попытки несанкционированного доступа блокируются и логируются.

## 6. Устранение неполадок и поддержка

### 6.1. Типовые ошибки и решения
| Ошибка | Причина | Решение |
|---|---|---|
| `Не удалось подключиться к ресурсу` | Ресурс недоступен, неверный порт/адрес | Проверить доступность ресурса, настройки сети, фаервол |
| `Ошибка аутентификации` | Неверный логин/пароль, блокировка учетной записи | Проверить учетные данные, статус пользователя, настройки LDAP/RADIUS |
| `Сессия завершена принудительно` | Превышение времени сессии, конфликт политик | Проверить политику доступа, расписание, лимиты подключений |
| `Не загружается интерфейс портала` | Ошибка службы веб-сервера, нехватка памяти | Перезапустить службы SafeConnect, проверить системные требования |

### 6.2. Сбор диагностических данных
Для обращения в техническую поддержку рекомендуется собрать:
- Логи системы: `/var/log/safeconnect/*.log`;
- Скриншоты ошибок;
- Версию ПО: раздел **Настройки → Система**;
- Конфигурацию сети и фаервола.

Данные можно экспортировать кнопкой **Собрать диагностику** в разделе **Настройки → Система**.
