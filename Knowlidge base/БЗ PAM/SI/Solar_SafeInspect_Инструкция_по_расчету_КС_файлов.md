# Общие сведения

В связи с тем, что при использовании для подсчета контрольных сумм стандартной для **Linux** программы **md5sum**, полученные контрольные суммы могут различаться в зависимости от использования диска (устройства) или образа (файла), рекомендуем Вам использовать входящую в состав **Astra Linux Смоленск 1.6 20.06 **программу **gostsum**. Помимо соответствия ГОСТ, использование gostsum гарантирует корректный подсчёт контрольных сумм независимо от использования с дисками или с образами.

На данной системе утилита **gostsum** присутствует в комплекте версии v2.2.5, и позволяет проводить расчет контрольных сумм с использованием алгоритмов, описанных в **ГОСТ 34.11.2012**.

Для проведения процедуры расчета контрольной суммы необходимо использовать следующую команду:

gostsum -r /folder/files/

Данная команда рекурсивно проверяет контрольную сумму всех файлов в указанной директории. Результат работы такой команды показан на рисунке (Рис. 1).

Таким образом, после проверки в терминале доступны контрольные суммы файлов из указанной директории.

Для облегчения выполнения контрольных процедур, можно использовать специальные скрипты, подготовленные для выполнения контроля исполняемых файлов Solar SafeInspect.

# Процедура выполнения контрольной проверки

Процедура выполнения контрольной проверки выглядит следующим образом:

1. Выполнить проверку контрольных сумм файлов с контрольными значениями в системе контроля целостности Solar SafeInspect. Для этого в Solar SafeInspect перейти в раздел **Система → Контроль целостности**. Нажать кнопку **Хэш-суммы **(Рис. 2).
![](Solar_SafeInspect_Инструкция_по_расчету_КС_файлов_media\image002.png)

Рис. 2. Контроль целостности

После выполнения данной процедуры станет доступен документ **executables-pdf**, в котором содержатся рассчитанные контрольные значения целостности файлов. Необходимо сохранить этот файл в директорию /home/test/загрузки/ (или /home/user/загрузки/) в виде файла **document.pdf** (Рис. 3).

![](Solar_SafeInspect_Инструкция_по_расчету_КС_файлов_media\image003.png)

Рис. 3. Документ «executables-pdf»

2. Установить, а затем подключить **Astra Linux Смоленск 1.6 20.06** на отдельную виртуальную машину.
3. Установить сетевой адрес** **таким образом, чтобы менеджмент интерфейс Solar SafeInspect был в той же сети, что и интерфейс установленной **Astra Linux Смоленск 1.6 20.06**.
4. Проверить подключение по SSH.
5. Используя указанную команду, добавить ключ для подключения по SSH к Solar SafeInspect, для работы скрипта проверки контрольных сумм:
ssh-copy-id -p 4772

где 4772 - это порт для подключения к Solar SafeInspect, admin - пользователь для подключения к Solar SafeInspect, 192.168.15.205 – адрес системы Solar SafeInspect.

6. Затем воспользоваться специальным файлом-скриптом, который сравнит контрольные суммы, рассчитанные в Solar SafeInspect с контрольными суммами, которые рассчитывает утилита **gostsum**.
7. Сохранить утилиту **si_executables_verifier.py** (данная утилита использует gostsum системы, подключается к Solar SafeInspect удаленно по протоколу SFTP, и сравнивает контрольные суммы файлов, указанные в списке хэш-сумм со значениями, рассчитанными системой Solar SafeInspect) в директорию, в которую ранее сохранили файл document.pdf.
8. Теперь в терминале системы выполните команду перехода в эту директорию:
cd /home/test/загрузки

9. Выполните скрипт **si_executables_verifier.py** со следующими параметрами:
si_executables_verifier.py -f document.pdf -u admin -a 192.168.14.45 –p 4772

Описание значений для подстановки:

si_executables_verifier.py -f pdf-файл с контрольными суммами из Solar SafeInspect -u пользователь, администратор Solar SafeInspect -a - адрес Solar SafeInspect -p - порт для подключения

При выполнении скрипта верные сравнения выделяются зеленым цветом (Рис.4), неверные суммы выделяются красным.

![](Solar_SafeInspect_Инструкция_по_расчету_КС_файлов_media\image004.png)

Рис.4. Итоговые сравнения

После выполнения скрипта выводится статистика по проверенным контрольным суммам. Таким образом, все контрольные суммы, подсчитанные внутренней утилитой системы Solar SafeInspect, совпадают с суммами, рассчитанными утилитой gostsum на Astra Linux Смоленск 1.6 20.06.
